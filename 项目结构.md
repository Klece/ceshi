#### 3.1.2 关键函数详细逻辑

**`load_all_configs()`** — 配置加载器
1. 尝试读取 `config/config.json`
2. 如果文件不存在，从 `default/default_config.json` 复制
3. 如果 default 也不存在，硬编码生成默认 config 对象并写回文件
4. 加载 `skills.json`（不存在则空列表）
5. 加载 `projects.json`（不存在则空列表）
6. 加载 `pixel-text.json`（不存在则硬编码默认数据）
7. 在模块加载时立即执行一次

**`make_github_request(url)`** — GitHub API 请求
- 关闭 SSL 验证（`verify=False`，用于Android环境）
- 超时 5 秒
- 返回 requests.Response 对象
- 失败时返回 MockResponse（status_code=500）

**`get_github_projects()`** — 获取 GitHub 项目
1. 从 config 获取 github_url → 提取 username
2. 调用 `/users/{username}/repos?sort=pushed&per_page=100`
3. 解析返回数据为项目列表格式
4. 限制返回最近 10 个仓库
5. 失败返回空列表

**`get_github_contributions(months=12)`** — 获取真实贡献数据
- 使用 `_contrib_cache` 全局缓存（一次请求/会话有效）
- 第一步：调用 `/users/{username}/repos` 获取所有仓库
- 第二步：对每个仓库调用 `/repos/{username}/{repo}/commits?author={username}&per_page=100`
- 第三步：用 SHA 去重（`processed` set）
- 第四步：调用 `/users/{username}/events/public?per_page=300`（补EventsAPI数据）
- 第五步：按月份聚合（1月-12月格式）
- **绝不生成假数据**，任何失败都返回全零

**`@app.route('/')`** 主页路由 — 数据聚合中心
- 读取 pixel-text.json
- 调用 get_skills_data_for_chart()、get_projects()、get_tech_stack()
- 调用 get_poems()、get_tooltips()、get_github_contributions()
- 检测背景图片存在状态
- 打包为 `github_info` 字典 → `render_template('index.html', ...)`

#### 3.1.3 数据流（从请求到页面）

```
浏览器请求 / → index() 路由
  ├─ 读取 pixel-text.json（像素文字）
  ├─ get_skills_data_for_chart()（技能图表数据，目前 skills.json 不存在→空）
  ├─ get_projects()
  │   ├─ get_github_projects()（GitHub API）
  │   └─ projects.json（API失败时备用）
  ├─ get_tech_stack()（config.json → tech_stack 数组）
  ├─ get_poems()（poems.txt → 字符串列表）
  ├─ get_tooltips()（tooltips.json → 对象）
  ├─ get_github_contributions()（贡献统计→12个月数组）
  └─ get_background()（检测背景图片是否存在）
       ↓
github_info = {avatar_url, name, bio, readme_content, recent_repos, 
               skill_data, skill_labels, skill_colors, tech_stack, 
               poems, tooltips, pixel_text, contributions}
       ↓
render_template('index.html', github_info=github_info, config=config, ...)
       ↓
Jinja2 渲染 → HTML → 返回浏览器
```

#### 3.1.4 服务端启动流程

1. 获取当前PID（`os.getpid()`）
2. 读取 `.flask.pid` 文件中的旧PID
3. 如果旧PID存在且不等于当前PID → `os.kill(旧PID, 9)` 杀旧进程
4. 写入当前PID到 `.flask.pid`
5. `app.run(debug=False, host='0.0.0.0', port=5000)`

### 3.3 `src/build_static.py` — 静态构建脚本

#### 3.3.1 执行流程

```
1. 读取 config.json（获取背景图片文件名）
   - 如不存在，从 default/default_config.json 复制
2. 动态导入 app.py（importlib）
3. 使用 Flask test_client 模拟 HTTP 请求生成所有页面：
   a. GET / → index.html（depth=0, 相对路径修复）
   b. GET /articles → articles/index.html（depth=1）
   c. GET /article/<slug> → articles/<slug>/index.html（每篇文章，depth=2）
4. 路径深度修复（fix_paths）：
   - depth=0(root): /fonts/ → ./fonts/, /article/slug → articles/slug/
   - depth=1(articles/): /fonts/ → ../fonts/, /article/slug → ./slug/
   - depth=2(articles/slug/): /fonts/ → ../../fonts/, /article/slug → ../slug/
5. 复制背景图片到 content/
6. 复制 fonts/ 目录（如不存在）
7. 写入 .nojekyll（GitHub Pages 兼容）
```

#### 3.3.2 用途

- 生成完整的静态HTML站点，无需Flask即可部署
- 所有数据在构建时一次性请求并写入HTML
- 生成 3 种页面类型：首页、文章列表、文章详情
- 自动修复所有资源路径为相对路径，适配 GitHub Pages
- 部署到 GitHub Pages 后，用户访问的是生成的静态文件

#### 3.3.3 注意事项

- `index.html` 和 `articles/` 目录在 `.gitignore` 中被忽略，需手动 `git add -f` 或解除忽略
- 背景图片和字体文件在项目根目录中已存在，构建时不会重复复制
- 构建时所有GitHub API请求会实时调用，API受限时可能得到空数据

## 七、已知问题清单

1. **`__pycache__/` 忽略不完整**：`.gitignore` 只写了 `__pycache__/` 无 `src/__pycache__/`
2. **skills.json 缺失**：app.py中引用了此文件但仓库中不存在，模板中也未使用相关数据

5. **Events API 与 repos API 数据重叠**：Events API 的 commits 没有 SHA 去重，可能少量重复计数
6. **文章详情页 Markdown 图片路径**：文章内容中的相对图片路径（如 `content/image.png`）在静态页中的解析路径与文章目录深度有关，可能导致图片无法加载

### 配置修改后生效方式
- 修改 `config/config.json` → 刷新页面即可（Flask 每次请求重新读取）
- 修改 `content/Introduction.md` → 刷新页面即可
- 修改 `templates/index.html` → 需重启 Flask 服务（开发中会检测到变化自动重启，如果 Flask 开启了 `debug=True`）
